## Esercizio: Il Validatore di Sintassi HTML Semplificato

### Contesto

In molti editor di testo e IDE, esiste una funzionalità che controlla se le parentesi o i tag sono bilanciati correttamente. La struttura dati ideale per gestire questo tipo di annidamento (LIFO - Last In, First Out) è lo **Stack**.

### Obiettivo

Scrivere un programma in Java che analizzi una stringa contenente una serie di tag HTML semplificati e determini se la sintassi è valida.

### Specifiche Tecniche

1. **Classe:** Crea una classe chiamata `HtmlValidator`.

2. **Metodo:** Implementa un metodo statico con la seguente firma:

```java
public static boolean isValid(String input)
```

3. **Input:** Il metodo riceve una stringa che simula del codice HTML. Per semplicità, assumiamo che i tag non abbiano attributi (es. solo `<div>`, non `<div class="row">`).

4. **Logica dello Stack:**

- Quando incontri un **tag di apertura** (es. `<html>`, `<div>`, `<b>`), devi inserirlo nello Stack.

- Quando incontri un **tag di chiusura** (es. `</html>`, `</div>`, `</b>`), devi verificare se corrisponde all'ultimo tag aperto presente in cima allo Stack.

- Il testo normale che si trova tra i tag (es. "Ciao Mondo") deve essere ignorato ai fini della validazione.

### Regole di Validazione

La stringa è considerata **valida** (`true`) se:

1. Ogni tag di apertura ha un corrispondente tag di chiusura.

2. I tag sono chiusi nell'ordine corretto (l'ultimo aperto è il primo a dover essere chiuso).

3. Alla fine dell'analisi, lo Stack è vuoto.

La stringa è considerata **non valida** (`false`) se:

1. Si incontra un tag di chiusura ma lo Stack è vuoto.

2. Si incontra un tag di chiusura che non corrisponde al tag in cima allo Stack (es. provo a chiudere `</b>` ma in cima c'è `<div>`).

3. Al termine dell'analisi, rimangono ancora tag aperti nello Stack.

### Esempi di Test Case

| **Input String** | **Risultato Atteso** | **Motivo** |
|---|---|---|
| `"<html><body></body></html>"` | `true` | Perfettamente bilanciato. |
| `"<html><body>Test</body></html>"` | `true` | Testo ignorato, tag bilanciati. |
| `"<b><i>Grassetto Corsivo</i></b>"` | `true` | Ordine corretto. |
| `"<html><body></html></body>"` | `false` | Errore di incrocio: `body` chiuso dopo `html`. |
| `"<div><p>Test</div>"` | `false` | Manca la chiusura di `<p>`. |
| `"<html><body>"` | `false` | Tag non chiusi alla fine. |
| `"</div>"` | `false` | Chiusura senza apertura precedente. |

### Requisiti Opzionali (Bonus)

- Gestisci il parsing della stringa in modo da isolare i tag (puoi usare `String.split`, `Regex` o un ciclo manuale sui caratteri `<` e `>`).

---

### Suggerimento per l'inizio

Non preoccuparti subito di come estrarre le sottostringhe (il parsing). Concentrati sulla logica dello Stack:

> *"Se inizia con `</` è di chiusura, altrimenti se inizia con `<` è di apertura. Se è di chiusura, controlla chi c'è in cima alla pila..."*
